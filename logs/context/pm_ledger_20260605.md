# ledger ドメイン実装タスク（フェーズ2）

## タスク概要
- 対象REQ-ID: REQ-LEDGER-001（仕訳入力）, REQ-LEDGER-002（勘定科目管理）, REQ-LEDGER-003（仕訳帳・総勘定元帳）
- 変更種別: 新規実装
- 概要: 複式簿記の中核機能。API（/api/ledger/**）と画面5つ（/journal-entries 一覧・new・[id]/edit、/accounts、/books/journal、/books/general-ledger）を実装する

- blockedBy: なし（共通基盤はmainにマージ済み）

## 実装ガイド
- API: src/app/api/ledger/accounts/route.ts, accounts/[id]/route.ts, journal-entries/route.ts, journal-entries/[id]/route.ts, general-ledger/route.ts
- バリデーション/サービス: src/lib/ledger/（zodスキーマ、貸借一致検証、FiscalPeriod解決、残高計算）
- 画面: src/app/(app)/journal-entries/**, accounts/, books/journal/, books/general-ledger/
- 仕訳入力フォーム: 明細行の動的追加/削除、科目セレクト（コード+名称）、借方/貸方入力、リアルタイム貸借差額表示、取引先セレクト、貸借不一致時は保存ボタン無効
- 一覧画面: 期間/科目/取引先/キーワードフィルタ、Pagination 使用
- 総勘定元帳: 科目セレクト必須、期首残高行+明細+累計残高列、期間フィルタ
- テスト: 貸借一致検証・期間解決・元帳残高計算のユニットテスト（Vitest、src/lib/ledger/*.test.ts）
