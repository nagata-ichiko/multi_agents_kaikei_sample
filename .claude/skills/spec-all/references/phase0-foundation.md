# フェーズ 0: 統一基盤の確立

Spec 執筆を始める前に、統一ルールをファイルに永続化する。
**PM のコンテキストが圧縮されても、各バッチがこのファイルを読めば統一性を維持できる。**

### 0.1 用語辞書の作成

1. コードリポの主要モデル名・コントローラ名を一覧化する
2. それぞれに対応する日本語用語を決定する
3. `docs/requirements/glossary.md` に「用語 / 定義 / コード上の表現 / 使わない表記」のテーブル形式で書き出す
4. ユーザーに確認する：「この用語統一で進めますか？」

### 0.2 統一基盤の確認

統一基盤確認チェックリスト（`.claude/rules/pm-orchestration.md` で自動ロード）を実施する。
特に [Spec 執筆標準](../../_shared/spec-writing-standard.md) のセクション構成・記述ルール・REQ-ID 規則を把握する。

### 0.3 REQ-ID カテゴリの事前採番

overview.md の全機能に対し、REQ-ID のカテゴリ部分を事前に決定する：

```
ユーザー認証・登録 → REQ-AUTH
ユーザープロフィール → REQ-PROFILE
案件管理 → REQ-PROJECT
...
```

これを `logs/context/specall_reqid_plan.md` に記録する。
各バッチはこのファイルを参照して REQ-ID を採番する。
