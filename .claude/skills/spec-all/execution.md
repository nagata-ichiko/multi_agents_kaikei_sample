## 分析エージェントへの指示テンプレート

[サブエージェントタスク指示フォーマット](subagent-task-format.md) に従い、
**全ドメイン（大規模分割後のサブドメイン含む）のサブエージェントを同時に並列起動する**。

**サブエージェントのコンテキスト枯渇対策:**
- 各サブエージェントに「最後に必ず完全な分析レポートを出力すること（途中で終わらない）」と明示する
- テーブル内の各行は簡潔に（1行80文字以内目安）し、出力トークンを節約する
- フェーズ1.2 の大規模ドメイン分割ルールにより、1エージェントあたりの負荷を事前に軽減する

各サブエージェントへの指示:


## フェーズ 4: 設計書統合更新

全バッチ完了後、基本設計書を最終更新する。
**Phase 3で大量のSpecを書いた後はコンテキストが逼迫している。** 記憶に頼らず、必ずサブエージェントで情報を再収集してから設計書を更新すること。

### 4.1 情報収集エージェントの起動

設計書ごとに必要な情報を全Specから抽出するサブエージェントを**並列起動**する：

```
エージェント1: 画面遷移情報の収集
  docs/requirements/features/*.md の全ファイルから「画面遷移図」セクションと
  画面テーブル（画面名・パス・コード根拠）を抽出してください。
  **機能横断の遷移も抽出すること:** 各Specの画面遷移図で遷移先が他のSpecの画面を指している場合
  （例: ユーザー詳細→ポイント履歴）、「機能横断遷移一覧」として別テーブルにまとめる。
  コードのルーティング定義・リンク先URLを根拠に、実在する遷移のみ記載する。
  出力: 公開サイト / マイページ / 管理画面 に分類した統合画面一覧と遷移一覧 + 機能横断遷移一覧

エージェント2: APIエンドポイント情報の収集
  docs/requirements/features/*.md の全ファイルから API エンドポイント情報
  （メソッド・パス・説明・コード根拠）を抽出してください。
  出力: 名前空間ごとに分類した統合エンドポイント一覧

エージェント3: データモデル情報の収集
  docs/requirements/features/*.md の全ファイルから「データモデル」セクションの
  テーブル情報（テーブル名・主要カラム・用途）を抽出してください。
  出力: カテゴリごとに分類した統合テーブル一覧
```

### 4.2 設計書ごとに更新・コミット

サブエージェントの結果をもとに、**1ファイルずつ更新してコミットする**。
途中でセッションが切れても成果が残るようにする。

更新順序と完了チェックリスト：

**1. `docs/design/screen-flow.md`**
- [ ] 全Specの画面遷移図セクションの情報が統合されている
- [ ] 公開サイト・マイページ・管理画面の3セクションに分類されている
- [ ] 各画面にパスとコード根拠が付与されている
- [ ] Mermaid図が[画面遷移図ルール](screen-transition-diagram.md)に準拠している
- [ ] 暫定版注記が削除されている
- → コミット: `feat: screen-flow.md を全Specと整合`

**2. `docs/api/openapi.yaml`**
- [ ] 全Specのエンドポイントが網羅されている
- [ ] 名前空間（admins/users/public/internal）ごとに分類されている
- [ ] 暫定版注記が削除されている
- → コミット: `feat: openapi.yaml を全Specと整合`

**3. `docs/design/db-design.md`**
- [ ] 全Specのデータモデルが反映されている
- [ ] ER図が主要テーブルの関連を正しく表現している
- [ ] 暫定版注記が削除されている
- → コミット: `feat: db-design.md を全Specと整合`

**4. `docs/design/architecture.md`**
- [ ] Specで判明した外部連携（WordPress等）が追記されている
- [ ] 非同期処理の用途が全て列挙されている
- [ ] 暫定版注記が削除されている
- → コミット: `feat: architecture.md を全Specと整合`

**5. `docs/design/shared-components.md`**
- [ ] 全ドメインの共通コンポーネントが集約されている
- [ ] 暫定版注記が削除されている
- → コミット: `feat: shared-components.md を全Specと整合`
