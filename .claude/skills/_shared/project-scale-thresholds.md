# プロジェクト規模の閾値定義

## 規模チェック方法
```bash
find . -type f -not -path './.git/*' | wc -l
```

## 閾値一覧

| 閾値 | 単位 | 用途 | 参照スキル |
|------|------|------|-----------|
| 200 | ファイル | コード分析の前処理要否判定。超過時は analyze-codebase を先に実行 | analyze-codebase, init-spec |
| 500 | ファイル | 骨格分析の閾値。超過時はサンプリング分析に切り替え | init-spec |
| 3,000 | 行（1ファイル） | ドメイン分割の目安。超過するファイルは分割を検討 | spec-all |
| 6 | 機能/ドメイン | 1ドメインあたりの機能数上限。超過時はドメイン分割を提案 | spec-all |
| 20 | Spec数 | フラット管理 vs カテゴリ分類の切替点。超過時はカテゴリ別ディレクトリに整理 | doc-accuracy |

## 閾値: 200ファイル（コード分析）

200ファイルを超えると、単一エージェントが全ファイルを読みながら横断的な関連を把握する精度が低下する転換点となる。

## 判定ロジック
| ファイル数 | analysis_*.md の有無 | アクション |
|-----------|---------------------|-----------|
| ≤200 | - | 前処理不要。そのまま続行 |
| >200 | なし | analyze-codebase の実行を提案 |
| >200 | あり | analyze-codebase 完了済み。結果を読み込んで続行 |
