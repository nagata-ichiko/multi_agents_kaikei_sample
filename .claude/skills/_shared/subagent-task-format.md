# サブエージェントタスク指示フォーマット

サブエージェントにタスクを委任する際の標準フォーマット。
analyze-codebase や他のスキルでサブエージェントを使う場合に適用する。

## 指示テンプレート

```
あなたは [役割名] です。[対象] を分析/実行してください。

## 対象
[対象のパスまたは範囲]

## 出力先
[logs/context/ 配下のファイルパス]

## レポート形式
### [セクション1]
[期待する内容]

### [セクション2]
[期待する内容]

## 厳守事項
- コードに書いていないことは書かない
- 推測・一般論は書かない
- ファイル名:行番号 or 関数名で根拠を示す
```

## 共通ルール
- サブエージェントはコードを変更しない（読み取り専用）
- レポートには必ずコード根拠を含める
- 不明瞭な箇所は「要確認」として報告する

## コンテキスト予算管理
サブエージェントへのタスク配布時は、コンテキスト消費を最小化する:
- **必要最小限のファイルリスト**: 対象ファイルを明示的にリストアップし、「全部読んで」は禁止
- **出力上限の指定**: レポートの最大行数を指示に含める（目安: 1タスクあたり200行以内）
- **段階的読み込み**: ファイル数が20以上の場合、まずファイル一覧とシグネチャだけを収集し、詳細分析は2回目のサブエージェントに委任する
- **重複排除**: 複数サブエージェントに同じファイルを読ませない。共通ファイルの情報はコンテキストファイルに事前抽出して配布する
