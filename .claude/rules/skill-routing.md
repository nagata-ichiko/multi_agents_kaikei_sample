# スキルルーティング

## 原則

ユーザーからリクエストを受け取ったら、**内容を自分で解釈して作業を始める前に**、まずスキル一覧とのマッチングを行う。

URL・ファイルパス・Issue番号など外部参照が含まれるリクエストは、その中身を取得する前の段階でスキルマッチを判定する。外部参照の内容はスキル内で確認すればよい。

## 判定フロー

1. リクエストのキーワード・意図をスキル一覧のトリガー条件と照合する
2. **特定スキルに確定できる場合**はそのスキルを直接発火する（例: 「Specを作って」→ spec-feature）
3. **内容が不確定な場合**（URL・Issue番号のみ、「これやって」等）は **handle** を発火する。handle が内容を確認し、適切なスキルに振り分ける
4. マッチするスキルがなければ直接対応する

## 禁止

- URL や Issue の中身を自分で取得してからスキル判定すること。中身の確認はスキル内で行う
