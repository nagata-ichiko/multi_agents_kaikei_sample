# Gitワークフロー

## ブランチ戦略
- ベースブランチ: main
- Devブランチ: 複数機能をまとめる作業用ブランチ。命名規則・PR先はプロジェクトごとにCLAUDE.mdまたはCLAUDE.local.mdで定義する
- 機能ブランチ: feature/[Spec ID]（例: feature/REQ-AUTH-001）

## コミットメッセージ規約

| 段階 | プレフィックス | 例 |
|------|-------------|-----|
| 設計書作成 | `docs:` | `docs: REQ-XXX-NNN 要件定義書を作成` |
| 実装 | `feat:` | `feat: REQ-XXX-NNN CRUD APIの実装` |
| 実装自己レビュー | `refactor:` | `refactor: REQ-XXX-NNN コード品質改善` |
| レビュー対応 | `fix:` | `fix: REQ-XXX-NNN レビュー指摘対応` |
| UT（ユニットテスト） | `test(ut):` | `test(ut): REQ-XXX-NNN Usecaseのユニットテスト追加` |
| ST（システムテスト/E2E） | `test(st):` | `test(st): REQ-XXX-NNN E2Eテスト追加` |

形式: `[プレフィックス]: [Spec ID] [変更概要]`

## ブランチ状態の定義

機能別ブランチ対応表で使用する状態ラベル。

| 状態 | 意味 |
|------|------|
| `設計中` | 設計書を作成中 |
| `実装中` | コード実装中 |
| `自己レビュー中` | 実装完了後の自己レビュー・リファクタリング中 |
| `レビュー待ち` | レビュアーにレビューを依頼中 |
| `レビュー対応中` | レビュー指摘の修正対応中 |
| `UT中` | ユニットテスト作成・実行中 |
| `ST中` | システムテスト（E2E・手動テスト）作成・実行中 |
| `PR済み` | DevブランチまたはmainへのPR作成済み |
| `マージ済み` | Devブランチまたはmainにマージ完了 |
| `完了` | 全作業完了（ブランチ削除済み） |
