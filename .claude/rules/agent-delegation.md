# Agent 委任ルール

## 原則

**判断不要な作業は Agent(Sonnet) に委任する。** 本体（Opus）はコンテキスト単価が高いため、判断・調査・レビューに集中する。

## 委任する作業（判断不要）

- 実装プランが明確なコード修正（「上から順にやるだけ」で完了するもの）
- パターンが決まっている一括置換・修正（テストコマンドの統一、import 修正など）
- 独立した複数ファイルの同種修正

## 委任しない作業（判断が必要）

- バグの原因調査・デバッグ
- テスト失敗の分析と修正方針の決定
- レビュー（設計意図の理解が必要）
- ユーザーとの相談・方針決定

## 判断基準

迷ったら「この作業に実装プランが書けるか」で判断する。プランが書けて手順が明確なら委任、曖昧さが残るなら本体で実行。
